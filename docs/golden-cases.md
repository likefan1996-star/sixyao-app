# 标准案例盘校验说明

标准案例盘用于对照校验排盘引擎和解读输出，重点覆盖：

- 纳甲
- 世应
- 六亲
- 六神
- 月建、日辰
- 本卦、变卦、动爻
- 用神与结果页白话解读输出

## 文件位置

- 案例库：`data/golden-cases.js`
- 校验脚本：`scripts/verify-golden-cases.js`

## 状态说明

每个案例都有 `status`：

- `draft`：草稿案例，只验证结构和当前引擎可运行，不作为标准答案。
- `verified`：已由人工、书籍案例或多个可信排盘工具交叉确认，脚本会对 `expected` 做严格比对。

不要把当前程序输出直接复制成 `expected`，除非已经完成外部校验。否则只是用程序证明程序自己正确，没有校验价值。

## expected 字段建议

确认后可以逐步填写：

```js
expected: {
  hexagram: '同人',
  changedHexagram: '家人',
  binaryStr: '101111',
  changedBinaryStr: '101101',
  shi: 3,
  ying: 6,
  dayStem: '戊',
  dayBranch: '辰',
  monthBranch: '巳',
  najia: ['己卯', '己丑', '己亥', '壬午', '壬申', '壬戌'],
  liuqin: ['父母', '兄弟', '官鬼', '父母', '子孙', '妻财'],
  liushen: ['青龙', '朱雀', '勾陈', '腾蛇', '白虎', '玄武'],
  movingPositions: [4],
  yongshen: '世爻',
  tierKey: 'xiaoji',
}
```

不需要一次填满。可以先确认本卦、变卦、世应，再逐步增加纳甲、六亲、六神和解读字段。

## 推荐推进节奏

第一批先确认 10 个：

- 2 个静卦
- 2 个单动爻
- 2 个多动爻
- 2 个纯卦、游魂、归魂相关案例
- 2 个节气边界案例

第二批扩到 20-30 个，覆盖感情、工作、财务合作、其他主题。

第三批扩到 50 个，再重点看解读输出是否稳定、易懂、不过度承诺。
